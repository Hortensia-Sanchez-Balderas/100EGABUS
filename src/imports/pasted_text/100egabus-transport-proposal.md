Sistema Integral de Gestión y Optimización del Transporte 100EGABUS
1. Introducción

El transporte gratuito “100EGABUS”, implementado por el municipio de Ciénega de Flores, Nuevo León, representa una iniciativa de gran impacto social al facilitar el traslado de estudiantes hacia distintos puntos del área metropolitana. Sin embargo, el crecimiento en la demanda del servicio ha evidenciado limitaciones en su gestión operativa, particularmente en la organización de usuarios, control de rutas y análisis de datos.

La presente propuesta plantea el desarrollo de un sistema integral enfocado en la gestión de usuarios y el análisis de datos operativos, con el objetivo de optimizar la toma de decisiones, mejorar la eficiencia del servicio y facilitar la planeación estratégica del programa.

2. Planteamiento del Problema

Actualmente, el programa presenta las siguientes problemáticas:

La gestión de estudiantes se realiza mediante hojas de cálculo (Excel), lo que limita la actualización, organización y análisis de la información.
No se cuenta con un registro estructurado que relacione a los usuarios con las paradas donde abordan el transporte.
La comunicación operativa depende de un grupo de WhatsApp, el cual presenta saturación y limitaciones funcionales.
No existe un sistema que permita analizar datos como:
Demanda por parada
Uso por horario
Costos operativos (gasolina)
Eficiencia de rutas

Estas limitaciones dificultan la toma de decisiones informadas y la optimización del servicio.

3. Justificación

El desarrollo de un sistema de gestión y análisis permitirá transformar la operación del programa 100EGABUS de un esquema manual a uno basado en datos.

Esto generará beneficios como:

Mejor organización de los usuarios
Obtención de información precisa sobre la demanda
Reducción de costos operativos
Optimización de rutas y horarios
Generación de reportes para la administración municipal

Asimismo, permitirá una planeación más eficiente del servicio en periodos semestrales, adaptándose a las necesidades reales de los estudiantes.

4. Objetivos
Objetivo General

Desarrollar un sistema integral que permita la gestión de usuarios y el análisis de datos del programa 100EGABUS, con el fin de optimizar su operación y apoyar la toma de decisiones.

Objetivos Específicos
Diseñar una base de datos estructurada de estudiantes y paradas.
Implementar un sistema de registro y control de usuarios.
Recabar información sobre el uso del transporte.
Desarrollar un dashboard interactivo para visualización de datos.
Analizar la eficiencia de rutas, tiempos y costos.
Proponer mejoras en la logística del servicio.
5. Alcance del Proyecto

El sistema propuesto incluye:

Registro y gestión de estudiantes
Asociación de usuarios a paradas específicas
Almacenamiento de datos operativos
Generación de indicadores clave
Visualización mediante dashboard

No se contempla en esta etapa la implementación completa de una aplicación móvil para usuarios finales, enfocándose el proyecto en la optimización administrativa y logística.

6. Metodología

El desarrollo del proyecto se basa en el modelo de simulación y análisis de sistemas:

6.1 Definición del sistema

Se analizará el funcionamiento actual del transporte, identificando variables como:

Usuarios
Paradas
Rutas
Horarios
Consumo de recursos
6.2 Formulación del modelo

Se definirá un modelo de datos que represente:

Relación entre estudiantes y paradas
Uso del transporte
Operación de unidades
6.3 Recolección de datos

Los datos se obtendrán mediante:

Observación del servicio
Registros existentes
Formularios de captura
Estimaciones operativas
6.4 Implementación

Se desarrollará el sistema utilizando herramientas tecnológicas para:

Almacenamiento de datos
Procesamiento de información
Visualización
6.5 Validación

Se compararán los resultados del sistema con datos reales y se evaluará su utilidad mediante pruebas piloto.

6.6 Experimentación

Se analizarán distintos escenarios, como:

Cambios en rutas
Ajustes de horarios
Eliminación o reubicación de paradas
6.7 Interpretación

Los resultados permitirán generar conclusiones para la mejora del servicio.

6.8 Documentación

Se elaborará documentación técnica y manual de usuario.

7. Propuesta Técnica

El sistema estará compuesto por tres módulos principales:

7.1 Módulo de Gestión de Usuarios

Permitirá:

Registrar estudiantes
Asignar paradas
Controlar vigencia del acceso
Mantener una base de datos actualizada
7.2 Base de Datos Centralizada

Almacenará información de:

Usuarios
Paradas
Horarios
Uso del servicio
Registros operativos
7.3 Dashboard de Análisis

Desarrollado en herramientas como Power BI, permitirá visualizar:

Usuarios por parada
Demanda por horario
Uso del transporte
Tiempos de recorrido
Consumo estimado de gasolina
Nivel de ocupación
8. Resultados Esperados
Identificación de paradas con alta y baja demanda
Optimización de rutas
Reducción de tiempos de traslado
Disminución de costos operativos
Mejora en la planificación semestral
Generación de reportes para toma de decisiones
9. Impacto del Proyecto

El sistema permitirá transformar la gestión del programa 100EGABUS hacia un enfoque basado en datos, mejorando tanto la operación diaria como la planeación estratégica.

Además, facilitará la generación de información clara y confiable para la administración municipal, contribuyendo a la mejora continua del servicio.

10. Conclusión

La propuesta presentada aborda las principales áreas de oportunidad del programa 100EGABUS mediante la implementación de un sistema integral de gestión y análisis.

Al centralizar la información y convertirla en indicadores útiles, se podrá optimizar el uso de recursos, mejorar la experiencia de los usuarios y fortalecer la toma de decisiones, garantizando la sostenibilidad y eficiencia del servicio en el largo plazo.
